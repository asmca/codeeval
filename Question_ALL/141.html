  • Flight 370

    Challenge Description:

    You've probably heard about the tragedy that happened with one of the Malaysian airplane. Many countries support the search mission with ships, planes and rescue services. Also there is a web app which allows all the people to help with the search. It's called Tomnod and you can read about it here. Thousands of volunteers are browsing the satellite images to explore the Earth and solve real-world problems. They mark the items they've found with a special tags, and all of the search results regarding the Malaysian flight end up in an XML file, which can be downloaded from here. Each item found and marked on the map is represented by a following XML section: 

         <Placemark id='4889783'>
		<name>Oil Slick ID 4889783</name><Snippet maxLines='0'></Snippet>
		<type>Oil Slick</type>
		<description><![CDATA[
			<img src='https://s3.amazonaws.com/explorationlab/chips/08458a6f42a780af66c05d0ef1385558.jpg' height=400 width=400><br/>
			<table width=400><tr><td width=40%>CrowdRank: <b>100%</b><br/>Confirmation: <b>2</b> people<br/>
			<td align='right' valign='bottom'>Image © DigitalGlobe 2014<br><br><a href='http://tomnod.com/nod/challenge/mh370_indian_ocean/map/983378'>See this point on <img src='http://www.tomnod.com/nod/images/logo_small.png' width=100></a>
		</tr</table>
		]]></description>
		<styleUrl>#tag_type_130</styleUrl>
		<TimeStamp><when>2014-03-27 18:14:25.234888</when></TimeStamp>
		<Point><coordinates>79.834979564799994,6.97011026235</coordinates></Point>
	</Placemark>

    Each item is called a placemark and it has it's "name", "type", "Point" with "coordinates", "TimeStamp" with "when" it was marked, and the "description" section with the "Confirmation" in it.

    In this challenge you're given a point on Earth surface represented by a longitude and latitude. Also you're given a radius in kilometers. You need to print out a placemark name which was confirmed by the largest amount of people in the given area.

    Input sample:

    Your program should accept as its first argument a path to a filename. The first N lines of the file contains the regions one per line. Then there will be the XML itself starting with the following line: 

    <?xml version='1.0' encoding='UTF-8'?>

    E.g.

    70; (96.289565663600001, 4.0044008079599998) 873; (92.749414588199997, 11.692508007600001) 98; (105.364957907, 6.6474719752600002) 20; (80.185393420699995, 7.5700396347199996) <?xml version='1.0' encoding='UTF-8'?> <kml xmlns='http://www.opengis.net/kml/2.2'> <Document> ... Document body ... </Document> </kml>

    Output sample:

    For each test case print out the "name" of a placemark which was confirmed by the largest amount of people. In case there is more than one placemark with the same maximum amount of confirmations, print the names ordered by "TimeStamp" starting from the most recent ones, separated by comma and a single whitespace. In case the "TimeStamps" are the same, print out placemarks in order of "Id". Print out 'None' in case there is no placemark in a given area. E.g.

    Other ID 4398192, Other ID 4402502, Other ID 4437554, Other ID 4437645, Oil Slick ID 4519894 Wreckage ID 6757595 Other ID 3835555 None

    Constraints:
    R is in range [20, 3000]
    N is in range [10, 15]
